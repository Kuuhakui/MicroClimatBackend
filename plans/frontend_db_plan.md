
# План Фронтенда и Базы Данных

## 1. Структура Фронтенда (React)

Приложение будет построено на базе React с использованием современного стека (Vite, React Router, React Query, Tailwind CSS, Lucide Icons).

### Компоненты и Страницы:
*   **Layouts**:
    *   `MainLayout`: Основной каркас приложения.
        *   **Sidebar (SidebarNav)**: Расположена слева. Содержит сгруппированные иконки для переключения между основными разделами (Dashboard, Map, Analytics, Events, Admin). Иконки должны иметь всплывающие подсказки (tooltips).
        *   **Header (TopBar)**: Верхняя панель. Содержит:
            *   Логотип и название текущего раздела.
            *   **Navigation Tabs**: Вкладки для переключения контекста внутри страницы (например, в Analytics: "Real-time", "History", "Reports").
            *   Блок пользователя (профиль, настройки) и индикатор уведомлений.
    *   `AuthLayout`: Экран входа.
*   **Pages**:
    *   `Dashboard`: Сводка по всем помещениям, критические алерты.
    *   `RoomMap`: Интерактивный план с SVG-подложкой.
        *   Маркеры датчиков (Live-данные через WebSocket/SSE).
        *   Индикация просроченной поверки.
    *   `Analytics`: Выбор датчика/комнаты, сравнение графика факта и прогноза.
    *   `Events/Journal`: Таблица событий, фильтрация по серьезности, кнопка "Взять в работу".
    *   `Settings (Core)`:
        *   Управление глобальными уставками (пороги по умолчанию).
        *   Конфигурация ML-моделей (выбор активной модели для комнаты).
        *   Управление пользователями и ролями (Admin only).
    *   `UserProfile`: Настройки темы, языка, уведомлений.

### Shared Components:
*   `StatusBadge`: Универсальный индикатор состояния.
*   `MetricCard`: Анимированная карточка с текущим значением и трендом.
*   `ChartContainer`: Обертка для графиков с поддержкой масштабирования.

## 2. Интеграция с API (Data Flow)

*   **Auth**: JWT в LocalStorage/Cookies, автоматическое обновление токена.
*   **Real-time**: Использование WebSockets для обновлений `sensor_measurements` и новых `system_events`.
*   **State Management**: `React Query` для серверного состояния (кэширование настроек, списков комнат).

## 3. Требования к данным (Core)

*   **System Settings**: Получение словаря настроек для динамической конфигурации UI.
*   **Thresholds**: Загрузка порогов для визуальной разметки графиков (красные/желтые зоны).
*   **Feature Flags**: Скрытие/отображение экспериментальных функций (например, "ML Forecast Preview").
